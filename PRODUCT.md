# Product

## Register

狗头军师 / Marshal — 一个狗军师，三个诸葛亮

## Users

**知识工作者**：作家、编辑、自媒体/内容运营、律师、产品经理、市场营销、研究员/分析师、学生、创业者等非开发者角色。他们使用 Claude Code 处理多个文档任务、分析工作、写作项目，希望同时驱动 3-5 个 agent 并行工作，坐在"统帅"位置调度协调。

关键特征：

* **不是程序员**，但会用终端工具完成专业工作

* 熟悉文件管理、多窗口切换，能看懂 Claude Code 的基本交互

* 同时处理多个独立项目（不同案件、不同文章、不同分析任务）

* 需要快速切换上下文，追踪多个 agent 的工作进度和文件改动

使用场景示例：

* 作家/编辑：多个 agent 同时进行不同章节的写作、资料整理、校对润色

* 自媒体/内容运营：并行处理公众号选题写稿、小红书文案、视频脚本

* 研究员/分析师：同时进行文献综述、数据分析、报告撰写

## Product Purpose

狗头军师/Marshal 是一个 **Windows 多 agent 指挥驾驶舱**（支持 macOS/Linux，但以 Windows 用户为主），让非技术用户能够在单个窗口内同时管理、监控多个 Claude Code agent 的并行工作。

产品解决的核心问题：

* **多 agent 并发控制**：顶栏 Tabstrip 快速切换 agent，支持同时查看多个 agent 的终端输出，无需在多个窗口间切换

* **上下文自动同步**：agent 与工作目录双向绑定，切换 agent 自动切换文件浏览目录，降低认知负担

* **文件浏览与编辑**：系统原生图标、列表/宫格视图、排序功能；Markdown 文件支持 WYSIWYG 编辑（Milkdown Crepe）

* **会话历史管理**：终端区「历史对话」下拉，查看并恢复某目录的所有 Claude 会话，快速续接上次对话

* **改动可视化追踪**：文件改动实时标记，跨 agent 跨项目的变更一目了然，不会遗漏任何改动

* **确认状态强提示**：agent 等待用户确认时通过多种方式（toast、闪烁、置顶）主动通知，即使用户在看其他 agent

* **舒适的视觉体验**：三套主题（暗色/护眼/亮色）适应不同光照环境和使用偏好，长时间工作不疲劳

成功标准：非技术用户能够流畅地指挥 3-5 个 agent 并行工作，不迷失在多个终端窗口中，能快速定位到需要注意的 agent 和文件。

## Brand Personality

**克制、大师、极简**

这是一个专业工具，设计语言应当体现**工匠品质**和**操作清晰度**：

* **克制**：不做装饰性的特效，每个视觉元素都服务于信息传达和交互反馈。用户是专业人士，不需要"有趣"的装饰来吸引注意力。

* **大师**：细节经得起长时间使用的推敲，间距、对齐、层级都有明确意图。像一把专业工匠的工具，手感精确。

* **极简**：去除一切非必要元素，让信息密度和可操作性达到平衡。用户需要快速扫视和决策，不是欣赏界面。

情绪目标：

* 给用户**掌控感**（我知道所有 agent 在做什么）

* 给用户**信心感**（改动清晰可追溯，不会丢失上下文）

* 给用户**专业感**（这是为严肃工作设计的工具，不是玩具或实验品）

## ~~Anti-references~~

明确**不要**的视觉方向：

1. **避免 SaaS 仪表板的平淡**：不要千篇一律的白底卡片、紫色渐变按钮、三列指标的模板感（Stripe Dashboard、Linear、Notion 等的中庸审美）
2. **避免游戏化/赛博朋克装饰**：不要过度的霓虹光晕、呼吸粒子、RGB 渐变、"黑客帝国"风格的终端装饰
3. **避免 Windows 10 风格的扁平**：不要完全无层次感的纯平面设计，需要保留适度的深度和边界感
4. **避免 macOS Big Sur 的圆润**：不要过度圆角（20px+）和毛玻璃效果的消费级产品感

具体反面案例：

* **不要**像 Vercel Dashboard 那样的极简主义到信息稀薄

* **不要**像 GitHub Copilot Chat 那样的对话气泡式布局

* **不要**像 VSCode 默认主题那样的中性灰到没有个性

* **不要**像 Windows Terminal 预览版那样的炫技特效（亚克力、云母材质）

## Design Principles

1. **为非技术用户设计专业工具**：用户是律师和作家，不是程序员。界面要直观到不需要查文档，但精确到不会误操作。避免"开发者自嗨"的晦涩交互。

2. **信息密度优先于呼吸感**：专业用户能处理更高的信息密度。多个 agent 并发时，屏幕空间是稀缺资源。紧凑但清晰，胜过宽松但需要滚动。

3. **状态即时可见**：不需要点击或悬停才能看到关键状态。agent 的运行状态、文件改动、等待确认等信息应该通过视觉层级主动传达。用户在处理案件/写作，不应该"找"状态。

4. **操作零歧义**：每个交互元素的功能边界清晰，不需要试探。按钮、tab、分隔条的可操作性一眼即知。误操作的代价可能是丢失工作内容。

5. **细节不妥协**：像素级对齐、一致的间距节奏、精确的颜色对比度。用户会长时间盯着这个界面工作，粗糙的细节会累积成疲劳。这是工具，不是展示品。

## Accessibility & Inclusion

**WCAG AA 级别**：

* 正文文字对比度 ≥4.5:1，大文字 ≥3:1

* 所有交互元素支持键盘导航（tab、arrow keys、快捷键）

* 所有动画和过渡效果响应 `prefers-reduced-motion: reduce`

* 颜色不是唯一的信息传达方式（状态用颜色+图标/文字组合）

已知用户需求：

* 长时间使用的专业工具，需要支持三套主题减轻视觉疲劳（暗色/护眼米色/亮色）

* 终端输出需要支持 Unicode 和中文路径（已实现 UTF-8 locale 配置）

* 多 agent 并发时，视觉区分要足够明显（不依赖细微色差）

* 用户可能在不同光照环境下工作（办公室/家中/咖啡厅），三套主题应覆盖不同场景

不追求 AAA 级别的超高对比度，因为：

* 终端输出本身的 ANSI 配色无法完全控制

* 过高对比度在暗色主题下可能造成视觉疲劳

* 平衡可读性和长时间使用的舒适度
